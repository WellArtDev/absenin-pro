# Decision Log — Absenin Product Brief

## 2026-06-08

### Intent
**Create** — new product brief for Absenin, a multi-tenant SaaS attendance platform.

### Key Decisions

1. **Multi-tenant from day one** — user explicitly chose SaaS multi-tenant over single-tenant. Architecture must support isolated tenant environments.

2. **Target audience**: mid-size companies (50+ karyawan) with mixed workforce (office + field sales + on-site technicians). Initial vertical: distribution/services.

3. **GPS modes**: three modes for sales — specific client location (verified), free mode, live location tracking. Not just radius-based check-in.

4. **Overtime rules**: HR-configurable per tenant — either auto-overtime (when work hours end) or checkout→overtime-checkin flow. Both modes supported in v1.

5. **Fingerprint integration**: Fingerspot selected as primary integration target — most common in Indonesia, Kerjoo already integrates with them. Other brands (ZKTeco, Solution) deferred to post-v1.

6. **Anti-spoofing v1**: selfie + GPS + device validation. Liveness detection and face recognition deferred to future versions.

7. **V1 scope**: all discussed features included — user said "masukin semua fitur". Payroll disbursement, BPJS, PPh 21, shift scheduling excluded from v1.

8. **Solo founder**: WellArtDev is building this as solo founder. Brief calibrated for fast execution — coaching mode skipped in favor of fast path.

9. **Document output language**: Indonesian (per project config).

10. **Mobile platform**: Android minimum for v1, iOS as stretch goal.
